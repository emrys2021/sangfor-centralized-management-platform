# 深信服 AC 上网行为管理统一管理系统

对多台深信服上网行为管理（AC）设备做集中管理：在一处维护多个实例的地址与凭据，统一查看、编辑并跨实例同步「自定义应用规则 / 自定义 URL 库 / 访问权限策略」，所有写操作留存审计日志。

接口层复用并改造自已验证的深信服 Web/CGI 与官方 API 客户端实现，复用其 RSA 登录 + CSRF 握手与各 CGI 报文逻辑。

## 功能

| # | 功能 | 说明 | 接口来源 |
|---|---|---|---|
| 1 | 自定义应用 | 仿原生管理表格（序号/规则名称/描述/应用类型/应用名称/状态/删除）；点规则名打开仿原生「编辑自定义应用」对话框；新增/编辑/删除 | `/cgi-bin/customrule.cgi` |
| 1b | 数据校验 | 自定义应用桑基图（策略→应用→协议端口→IP/域名）、自定义 URL 桑基图（策略→URL库→URL条目）、力导向应用关系图；策略→应用/URL 连线按动作着色（放行绿/拒绝红，多规则引用同对象时按「首条命中规则」为准）；私有条目按类型聚合、共享条目标红；两张桑基图可一键导出为**自包含离线 HTML**（ECharts SVG 渲染，矢量无极缩放、保留悬停高亮分支与提示框）；桑基图后附「**策略引用校验**」：列出全部访问权限策略及各自引用用户数，**无人引用（创建后没人用）的高亮**，可单条删除或一键清理（带预览清单+二次确认）；**若有组织组读取失败，「无人引用」结果不可信（失败组的用户没统计到、其在用策略会被误判无人引用），此时禁用「一键清理」并红字提示原因+列出失败组，要求先「重新校验」到 0 失败再清理，防止误删在用策略**；**「一键清理」在删除前会强制重新校验一次、用最新结果做删除依据（避免拿过期分析误删「已变成有人引用」的策略），刷新后变有人引用的会自动移出清理清单；策略删除后与增改一致失效该实例的分析/对比/搜索缓存，各视图不残留已删策略**；各分析带服务端 TTL 缓存（默认 5 分钟），「重新分析/校验」强制刷新 | 聚合 customrule + objurlgrp + netpolicy 详情；策略引用校验聚合 org.cgi 组织树/用户的生效策略（并把「与所属组相同」的继承占位展开为所属组策略，避免整组继承的策略被误报无人引用） |

| 3 | 访问权限策略 | 仿原生管理表格（复选框/序号/策略名称/适用用户/适用位置/适用目标区域/策略管理员/上移下移/过期日期/状态/删除，已过期标红）；上下移箭头调整顺序、勾选多行批量启用/禁用、行内删除（二次确认后均真实写入）；点策略名或「新增」打开**仿原生「访问权限策略」对话框（新建/编辑同一界面）**：启用该策略/名称/描述 + 「应用控制」规则增删改（每条经仿原生「选择适用应用」选择器设动作/生效时间/应用·URL，含自定义/内置应用与 URL） | `/cgi-bin/netpolicy.cgi` |

| 5 | 跨实例同步 | **单对象**：选源选目标预览差异 / 二次确认后真实写入（应用·URL 直写；策略据目标 crc 重映射，缺失的自定义引用自动先建；内置缺失会导致「降级（不等价）」策略，**真实写入默认拒绝、需勾选「允许降级同步」**，dry-run 始终预览）。**批量**：一次同步整类对象（全部应用/URL/策略），可选**镜像模式**删除目标上「源没有」的对象 | 组合上述接口 |

## 技术栈

- 后端：FastAPI + SQLAlchemy 2.0 + SQLite（凭据 Fernet 加密存储）
- 前端：React + Vite + TypeScript + Tailwind CSS + shadcn 风格组件（暗色主题以近纯黑中性面为主，发丝边框 + 单一克制紫强调色，参考 Linear / Vercel 一类风格）
- 列表：TanStack Table（列排序、分面筛选、表头固定）；可视化：ECharts 桑基图（懒加载）；数据请求：TanStack Query + axios

## 快速开始

### 后端

```bash
cd backend
uv sync                     # 安装依赖
cp .env.example .env        # 按需配置 SANGFOR_SECRET_KEY
uv run uvicorn app.main:app --reload --port 8000
```

打开 http://127.0.0.1:8000/docs 查看 API 文档。

> 所有环境变量统一使用 `SANGFOR_` 前缀（如 `SANGFOR_DEBUG` / `SANGFOR_DATABASE_URL` / `SANGFOR_CORS_ORIGINS` / `SANGFOR_REQUEST_TIMEOUT`），避免误读系统里的同名通用变量（如裸 `DEBUG`）。
>
> **并发限流**（两级，均按 `SANGFOR_` 前缀）：
> - `SANGFOR_FETCH_CONCURRENCY`（默认 8）：各分析 / 对比 / 搜索**单次请求**内并行拉取设备的线程池大小。
> - `SANGFOR_GLOBAL_FETCH_CONCURRENCY`（默认 16）：对**同一台 AC** 的**全局**并发上限（跨请求，按 `origin` 用信号量封顶）——无论同时多少请求、堆了多少线程，对单台设备的在途 CGI 请求数都不超过此值，避免高峰压垮设备。一般设为单请求上限的 1~2 倍，让少量并发请求不至于互相拖太慢。
>
> 注：限流是**进程内**的，若用多进程部署（多个 uvicorn worker），实际全局上限为 `worker 数 × SANGFOR_GLOBAL_FETCH_CONCURRENCY`。
>
> 生产环境务必在 `.env` 中设置固定的 `SANGFOR_SECRET_KEY`（Fernet key），否则换机后已存凭据无法解密。
> 生成方式：`uv run python -c "from cryptography.fernet import Fernet; print(Fernet.generate_key().decode())"`
> 密钥与加密时不一致（丢了 `data/secret.key`、迁移库漏带密钥、误换了密钥）时，连接会明确报错「凭据解密失败」，而不是误判成「未配置」或「用户名密码错误」。
>
> **鉴权**：设置 `SANGFOR_API_TOKEN` 后，所有 `/api` 接口（健康检查除外）要求请求头 `X-API-Token`（或 `Authorization: Bearer <token>`）与之匹配；前端在 `frontend/.env` 配置同值的 `VITE_API_TOKEN` 即可自动携带。未设置时不鉴权（仅限受信任内网，后端启动时会打印告警）。
>
> 与 AC 通信默认不校验 TLS（适配自签证书设备）；如需校验可设 `SANGFOR_VERIFY_TLS=true`（系统 CA）或指定 CA 证书文件路径。

### 前端

```bash
cd frontend
npm install
npm run dev                 # http://localhost:5050 （已配置 /api 代理到 8000）
```

> 若后端启用了 `SANGFOR_API_TOKEN`，在 `frontend/.env` 中设置同值的 `VITE_API_TOKEN`，前端会自动给所有请求附带 `X-API-Token` 头。

## 使用流程

1. 「实例管理」中新建实例，填写地址与凭据，点「测试」确认连通。
2. 顶部实例切换器选择当前实例，进入「自定义策略 / 自定义 URL / 访问权限策略」查看配置。切换器每个实例名前有连接状态圆点（绿=正常 / 红=连接失败 / 黄=未配置凭据 / 灰=已禁用），下拉展开时探测各实例、当前实例常驻探测；连接失败的实例仍可选中以便排查或重测（采用复用会话的轻量健康检查，不影响已登录会话）。
3. 「实例对比与同步」页按**意图**分两档（读写分离）：**对比（只读）** 与 **同步（写入）**。范围（单个对象 / 已选多个 / 全部对象）不是独立开关，而是**对象选择框里的子选项**：可**多选**具体对象，首项「全部对象（N 个）」与具体对象**互斥**（勾它会清掉其余单选，反之亦然）。选中 1 个走单对象详情（字段级 diff / 单对象写）；选中 2 个及以上（不论是手选的子集还是「全部对象」）都走批量渲染（汇总名单视图），已选子集时后端只按这几个名字拉取/处理，不用等全量——「对比名称和内容」这类原本要拉全部对象详情的慢操作，少选几个就能秒回。写入口只在「同步」档出现，「对比」档没有任何写按钮。
   - **对比 · 单个对象**：选源与对象，「开始对比」出字段级差异。自定义应用/URL 做字段级对比（IP/域名/URL 列表做行级增删、相同项弱化折叠）；访问权限策略的规则用**双栏 side-by-side**（左源右目标，类似 BeyondCompare）展示：底层用**加权序列对齐**（以引用重合度 Jaccard 为权重的序列对齐 DP，非交叉、两侧顺序单调）——完全相同=「一致」、最像的两条合并为「变更」、无共同引用的各自算「新增/目标多出」，**不会因一侧多插一条而后续全部错位、也不会把改动版规则错配到不相干的一条**；对齐的两条同显一行，一侧独有的对面**留白占位**，两列从上往下各自就是各自的真实顺序。每侧各自**高亮本侧独有**的引用/动作（源侧绿=目标没有、目标侧红+删除线=源没有、两侧共有=灰）；连续的「一致」规则超过 6 条会**折叠**成一条可展开提示，避免长策略刷屏。
   - **对比 · 多个对象（已选子集或全部）**：两个按钮各一种深度——**「只对比名称」** 只按对象名分 **仅源有 / 仅目标有 / 两边都有**（不拉详情、秒级）；**「对比名称和内容」** 逐对象比名单 + 内容，分 **仅源有 / 仅目标有 / 内容一致 / 内容不一致**，点开某条看明细（不一致→字段/规则级差异；仅源/仅目标→只读快照内容）。策略比对忽略各设备各自分配、跨实例必然不同的 rule_id/crc（避免把「内容相同」误判为不一致）；展开明细时规则用**双栏 side-by-side + 加权序列对齐**（引用重合度加权，非交叉、两侧顺序单调，类似 BeyondCompare）——完全相同=一致、最像的两条合并为变更、无共同引用的各自算新增/目标多出，两列各自保留真实顺序、一侧独有的对面留白，连续一致超 6 条折叠。**纯只读、不写设备**。内容对比的源/目标「该类全部对象快照」并行拉取并按 `(实例, 类型)` TTL 缓存（写操作后失效），首次较慢、之后秒回。
   - **策略对比范围（重要）**：访问权限策略的**对比/新建/显示只覆盖「应用控制 / 端口控制 / 代理控制」三段 + 策略启用状态**——应用控制比每条规则的**动作（放行/拒绝）**与引用应用/URL；端口控制比目的 IP 对象/服务/动作；代理控制比 http/sock/errorproto 开关；各段先比 include 开关，开关为关时不比该段规则。**其余段（Web 关键字/文件类型过滤、邮件过滤、QQ 号白名单、SaaS 高级选项等）不参与对比、也不在差异里显示**（接口能取到，只是当前有意省略）。故「同引用但动作相反 / 一条被禁用 / 端口控制不同」现在都会正确标为不一致，不再误判「一致」。
   - **同步 · 单个对象**：「同步预览」出 dry-run，确认后同步到所选或一键推送全部。
   - **同步 · 多个对象（已选子集或全部，批量）**：一次把源的这批对象（已选子集或全部应用/URL/策略）同步到目标。批量为**全量 upsert**：目标无同名→新增，已有同名→**覆盖**为与源一致（**不逐项比对内容**——要看内容差异请用「对比」档）。结果按目标分组显示新增/覆盖/删除/失败数与名单。**「同步预览」(dry-run) 只按名称分类、不逐个拉详情**，所以很快；真实写入才逐个建/改/删。可勾选**镜像模式**——删除目标上「源没有」的同类对象（仅自定义对象；内置 URL 库不受影响）；**镜像仅支持「全部对象」，选中已选子集时不可用**（否则会把未选中、但双方都合法存在的目标对象误删）。镜像开启时确认弹窗会红色警示、务必先「仅预览」核对删除清单。镜像真实删除有两道安全措施：**策略**删除前会对目标做一次强制刷新的引用校验，**在用（有用户引用）的策略跳过不删**、校验失败或校验结果里没有该策略名（无法确认是否在用）时该策略镜像删除也会被拒绝、不当作 0 引用直接删；且每个被删对象删除前都读取完整详情作为**删前快照**单独落审计，误删后可按快照重建。**引用校验只在真实执行时做**（遍历组织树较慢，放进预览会拖慢秒回）——策略镜像删除的预览是「候选名单」，真实执行时若某策略恰好在用，会被跳过而非删除，与预览可能不同（界面已提示）。
   - 「同步」档的写入都走二次确认弹窗，可选「仅预览（dry-run）」或「确认写入」；改动左侧配置后右侧结果不清空而是标记「已过期」并提示重新执行。
4. 「全局搜索」输入域名或 IP，反查当前实例中哪些自定义应用、自定义 / 内置 URL 库配置了它。智能匹配：通配条目 `*.deepin.org` 命中查询 `www.deepin.org`；查询父域 `deepin.org` 也能命中子域条目；IP 支持 CIDR 网段与 `a-b` 范围的区间重叠。结果按「自定义应用 / 自定义 URL 库 / 内置 URL 库」分组并标出命中的具体条目。索引需逐条访问设备构建（带 TTL 缓存），首次较慢、之后秒回；写操作后自动失效，亦可点「重建索引」强制刷新。
5. 「操作日志」查看所有写 / 同步动作的审计记录。每条都带可读「摘要」（如「新增规则：拒绝 → 钉钉、微信」「新增 URL 库『test』（2 条 URL/IP）：1.1.1.1、example.com」「新增自定义应用『app』（IP 2 条：…）」）；顶部搜索框对摘要 / 对象 / 操作人 / 实例 / 变更前后快照做子串检索——可按你添加过的 IP、域名、URL 等直接搜到相关记录。策略编辑的「变更前/后」存精简的「规则=动作+引用」快照而非整条原始报文；**删除类操作（自定义应用 / URL 库 / 策略的单删与镜像删除）真实执行前会读取完整详情存入「变更前」快照**，误删后可按快照内容重建。

## 写操作安全模式（重要）

读功能已完整可用。**写操作默认安全**：所有写入口（增删改、同步落盘）均支持 `dry_run`，仅构造并返回将提交给 AC 的报文、不改动设备；只有显式确认（开关 / 二次确认弹窗）后才以 `dry_run=false` 真实提交。未在 `CONFIRMED_WRITES` 白名单中的写操作即便传 `dry_run=false` 也会被拦截。

已确认并可真实写入：

- **自定义应用「新增(add)」「编辑(modify)」「删除(delete)」**（据真实抓包对齐报文结构）。新增/编辑在表单底部打开「实际写入设备」开关提交（默认关闭，仅预览）；删除点行内删除图标并二次确认后直接真实写入。
- **访问权限策略「编辑(modify)」**：点策略名打开仿原生「访问权限策略」对话框（与「新建」同一界面），可改启用该策略 / 描述、增删改「应用控制」规则（动作 + 引用的应用/URL）。采用「读—改—写」往返——后端先 `listItem` 取回完整策略对象，替换 `appctrl.application.data`（按规则名匹配，新增规则生成 ID、未提交的删除）与 include/enable/depict，其余配置（keyword/filetype/saas/mail/适用对象/高级配置…）原样保留，再以 `opr=modify` 提交。名称只读（按名匹配往返）；底部「实际写入设备」开关默认仅预览。
- **访问权限策略「上移(moveup)」「下移(movedown)」**（调整策略执行顺序，据真实抓包对齐）。策略列表「上移/下移」列的箭头点击后直接真实写入并刷新顺序（首/末行对应箭头置灰）。
- **访问权限策略「批量启用(enable)」「批量禁用(disable)」**（`name` 为名称数组，据真实抓包对齐）。策略列表勾选多行后，右上工具栏「启用/禁用」直接真实写入。
- **访问权限策略「新建(add)」**（`opr=add`，`data` 结构同 modify，默认骨架取自真实抓包）。「新增」与上面的「编辑」共用同一对话框，新建时名称可填（重名校验）。当前默认只建「访问权限策略」、只配「应用控制」；**适用用户不在新建报文中**（设备另存），新建后需在 AC 配置适用人员。
- **自定义 URL 库「新增(add)」「编辑(modify)」「删除(delete)」**（据真实抓包对齐报文：`data` 为 `{id,name,depict,url,keyword}`，`url` 为换行分隔的 URL/IP；删除 `name` 为名称数组）。在「自定义 URL 库」页点「新增 URL 库」或点开某自定义库后「编辑 / 删除」；新增/编辑有「实际写入设备」开关（默认仅预览），删除二次确认后真实写入。

- **访问权限策略「删除(delete)」**（据真实抓包对齐报文：`opr=delete`、`name` 为名称数组）。策略列表「删除」列点垃圾桶图标并二次确认后直接真实写入并刷新。

> **跨实例同步落盘**：自定义应用 / URL 库可真实写到目标实例（复用上述已确认报文）。在「实例同步」页点「同步到所选目标 / 一键推送到全部」会弹出二次确认，列出对象、源/目标实例，可选「仅预览（dry-run）」或「确认写入」；真实写入后自动失效目标实例的分析缓存。
>
> **访问权限策略的跨实例同步（含 crc 重映射 + 引用对象自动创建）**：apply 时预读源实例的**完整策略对象**与源自定义应用 / URL 名单，再据目标实例的应用目录树（`listAppTree`）按引用路径反查目标自己的 `crc`——解决「crc 由各设备独立分配、源 crc 在目标无效」的核心障碍。落盘走**已验证的写路径**：目标已有同名策略 → `modify_policy_application`（读目标自身策略为底座、仅替换规则的动作与引用，其余原样保留，与单实例编辑同一契约）；目标无此策略 → `create_policy`（`opr=add`，data 取自源对象、crc 重映射）。
> - **引用对象自动创建 / 更新 + 最大努力同步**：策略引用的对象若目标缺失，会区分「自定义」与「内置」——**自定义**应用 / URL 库复用已确认写接口同步到目标：**目标已有同名对象则改成与源一致（update），没有才新增（add）**，避免对已存在对象误走 add 报「名字已被使用」；同一对象被多个引用路径命中（如一个 URL 库的多个子类）只处理一次（去重）；含「自定义」前缀与应用名粘连成一段如 `自定义应用_AIGC应用` 的识别。缺失的**内置**对象无法创建（多因目标应用库版本较旧），跳过这些引用会写出与源**不等价（降级）**的策略（如拒绝规则少挡了对象=安全缺口）。为避免「悄悄改语义还报成功」：**dry-run 始终预览降级并列出被跳过的引用；真实写入默认拒绝降级**——内置缺失时拒绝发生在**创建任何引用对象之前**（`refused`、连被引用的自定义应用/URL 也不创建、完全不动设备）；**自定义引用对象若创建失败**（目标设备报错），也会导致引用缺失、**同样默认拒绝写策略**（此时可能已成功创建部分对象、保留在目标，消息会说明）。需在确认框勾选**「允许降级同步」**（`allow_degrade`）才写入降级版本，且结果徽标标为**「降级同步 / 已拒绝」**而非「成功」。
> - **逐步结果与排查**：策略同步涉及多个请求（建多个自定义引用 + 写策略），每个请求**独立**——单个自定义引用创建失败不拖垮整体（失败项单独记录、其引用按缺失跳过），结果卡片「逐步结果」逐条列出每步成功 / 失败与原因；写策略失败以失败态返回而非裸异常。底层接口失败信息带上 `path` / `opr` 与设备原始响应（设备常不给 `msg`），不再只显示「接口返回失败」。
> - 结果卡片可展开「查看重建报文（含目标 crc）」核对；适用用户不随报文跨实例，新建策略后需在目标手工配置适用人员。
> - **写入走已验证路径**：目标有同名策略 → `modify_policy_application`（读目标底座、仅替换规则）；目标无 → `create_policy`，且 data 由 `policy_template.build_policy_create_data`（真实抓包的最小骨架）+ 重映射规则构造，**不**回放源 listItem 的完整原始对象（源对象的异构/多余字段曾导致目标固件 `RemoteDisconnected` 断连）。
> - **新旧固件兼容**：新固件策略走 `netpolicy.cgi`，旧固件（如部分早期版本）走 `acnetpolicy.cgi`——首次访问策略时 `_ensure_policy_cgi` 自动探测并切换路径（读/写/列表统一），策略类型名「访问/上网权限策略」「SSL解密/上网审计策略」均兼容。旧固件 add 报文与新固件结构一致，仅 `ssl` 段需完整默认结构（据真实抓包，`create_policy` 自动补齐）；`acnetpolicy.cgi` 的 `add`/`modify` 已登记到写白名单。
> - 引擎见 [`app/services/policy_sync.py`](backend/app/services/policy_sync.py)：`build_app_index`（树→`path→{crc,type}` 索引，每目标建一次）、`build_remapped_rules`（重映射规则引用 crc）、`classify_missing`（缺失分「可自动创建/内置缺失」）、`ensure_referenced_objects`（建/改自定义引用，去重、目标已有则改成一致）、`sync_policy_to_target`（编排，逐步 `details` + `ok`）。差异预览（diff）对三类对象均可用。
>
> 同步差异预览会区分「目标不存在（可新增）」与「读取失败」：读取/解析失败时明确报错、不再误判为「将新增」，且 apply 阶段会跳过读取失败的目标、不会据此误新增。
>
> **访问控制**：真实写入入口（策略上下移、批量启停、各类增删改、同步落盘）默认不鉴权。部署到非完全可信网络前，请设置 `SANGFOR_API_TOKEN` 启用令牌校验（见上文「快速开始」）。

> 协议字段据抓包确认 TCP=0；UDP/ICMP 标签为推测，可按需调整 `PROTOCOL_OPTIONS`。
>
> **应用树与 crc**：策略里每个应用/网址以 `{path, type, crc}` 表示。`crc` 一律取自设备——「选择应用 / URL」选择器的应用树来自 `acnetpolicy.cgi` 的 `listAppTree`（[`web_client.list_app_tree`](backend/app/sangfor/web_client.py)，经 `GET …/policies/app-tree` 暴露），树中每个节点带 `name`/`type`（`catagory`/`app`）/`crc`/`value`（即引用 path），内置应用、自定义应用与「访问网站」URL 类目都在其中，选中即用其 `value`/`type`/`crc` 构造引用，**无需臆造 crc**。请求参数对齐 AC 原生对话框（`containFileType=false`、`containUrlType=false`）：此时「访问网站」下的各 URL 库（含「网站浏览/文件上传/其他上传/HTTPS」4 个子类）已**内联**在树中、点开无需再请求；注意 `containUrlType=true` 反而会清空「访问网站」子树。
>
> **策略引用校验（找出没人用的策略）**：设备无「策略→用户」反查接口，但**用户列表里每个用户带 `strategy`**（设备算好的生效策略：组默认 + 用户添加 − 排除）。故 [`org_cgi`](backend/app/sangfor/org_cgi.py) 用 `listorg.cgi` 取组织树、`org.cgi` 逐组取成员用户（`org:false` 行），[`policy_usage_service`](backend/app/services/policy_usage_service.py) 并行遍历所有组、把全体用户的 `strategy` 取**并集** = 被引用的策略；访问权限策略全集里不在并集内的即「无人引用」。因依赖设备自身的生效计算，**排除/组成员展开等边界都由设备处理、不会漏判**。结果带 TTL 缓存（`GET …/policies/usage`），写操作后失效。

通用启用真实写入的步骤：

1. 在 AC 控制台对某类对象做一次增/改/删，用浏览器开发者工具抓取请求的 URL 与 payload。
2. 据此核对/补全 [`backend/app/sangfor/web_client.py`](backend/app/sangfor/web_client.py) 中对应的 `create_* / update_* / delete_*` 报文。
3. 把已确认的 `(cgi_path, opr)` 登记到该文件顶部的 `CONFIRMED_WRITES` 集合。
4. 调用写接口时传 `dry_run=false` 即可真正提交；未登记的写操作会被拦截以防误写。

## 规则动作（允许/禁止）字段说明

访问权限策略详情中，每条规则的动作（允许/禁止）由 [`web_client.get_policy_detail`](backend/app/sangfor/web_client.py) 的 `extract_action` 从规则报文里按常见键名（`action`/`act`/`permit` 等）容错解析并归一化为 `allow`/`deny`/`unknown`。不同 AC 固件字段命名可能不同——若界面上动作显示为「未知」，打开该策略详情的「原始数据」标签，把规则里的动作字段名/取值告知，即可在 `extract_action` 中补全映射。
